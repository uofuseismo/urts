◆ setPollingTimeOut()

void URTS::Services::Standalone::Incrementer::ServiceOptions::setPollingTimeOut ( const std::chrono::milliseconds &  timeOut)

To receive replies we poll on a socket. After this amount of time has elapsed the process can proceed and handle other essential activities like if the program has stopped. If this is too small then the thread will needlessly burn cycles on a while loop but if its too big then thread will be unresponsive on shut down.

Parameters
[in]timeOutThe time to wait for a request before the thread checks other things.
Exceptions
std::invalid_argumentif this is negative.
